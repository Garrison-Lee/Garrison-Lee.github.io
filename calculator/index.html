About this app

This calculator allows for 5 button inputs. WASD are used to highlight the various buttons and "enter" can be used to select the currently highlighted button.

I decided to test a two button version ("next" and "enter") vs a five button version ("next" "previous" "up" "down" and "enter") to see which is the faster of the two versions. To test this, I timed how long it took -- averaged over 3 trials each -- to input the following equations:

  • (1+9)/8
  • (3-2+6)*2
  • (4/8)*4+7
  • 2+2

Five button layout:

  • Trial 1: 32 seconds
  • Trial 2: 36 seconds
  • Trial 3: 24 seconds
  • Average: 30.7 seconds

Two button layout:

  • Trial 1: 75 seconds
  • Trial 2: 65 seconds
  • Trial 3: 62 seconds
  • Average: 66.7 seconds

Errors: I didn't input any errors although I did "overshoot" the button I needed next several times. On the five button interface this can be remedied by simply hitting the select previous button, but when I was using the two button interface I had to iterate through all possible keys again which added significant time.

Conclusion:

Clearly we can see that the five button layout is superior to the two button layout. While it did save some time in that you could take a more direct path to the next button than just iterating through the list, the biggest time killer seemed to be that the five button layout allowed you to quickly fix the problem if you went past the button you needed. So if you need to hit 2 and you are currently on 3, it's one button press to go back if you have five buttons. If you are using the two button interface, it takes 15!
